* [ ] 系统资源

  ```
  一个系统通常有很多硬件资源和软件资源，归纳起来可以分为4类：处理机、存储器、I/O设备以及文件（数据和程序）
  ```

* [ ] 微机操作系统的发展

  ```
  1.单用户单任务操作系统：只允许一个用户上机，只允许用户程序作为一个任务运行，代表性CP/M、MS-DOS
  2.单用户多任务操作系统：只允许一个用户上机，但允许用户将程序分为若干个任务，允许他们并发执行
  3.多用户多任务操作系统：允许多个用户通过各自的终端，访问同一台机器，共享主机系统的各种资源。
  ```

## 操作系统的基本特性

* [ ] 特性

  ```
  操作系统总共有并发、共享、虚拟和异步四个基本特征。
  
  ```

* [ ] 并发和并行

  ```
  并发和并行相似，又有区别。
  并行是多个事件在同一时间内运行，
  并发是多个事件在同一时间间隔内运行。
  ```

* [ ] 进程简述

  ```
  所谓进程，是指在系统中能够独立运行并作为资源分配的基本单位，他是由一组机器指令、数据和堆栈组成的。多个进程之间可以并发执行和交换信息。进程和并发是现代操作系统中最重要的两个概念。
  ```

* [ ] 共享

  ```
  共享是指系统中的资源是由系统中正在执行的进程并发使用。目前实现共享的方式用两种。
  1.互斥共享，2.同时访问。
  
  互斥共享：一段时间内，只允许一个用户访问该资源，这种一段时间内只允许一个进程访问的资源，被称为临界资源。系统中的大多数物理设备、栈、变量和表格都属于临界资源。
  同时访问：允许多个进程同时访问，典型的类似的资源是磁盘。
  ```

* [ ] OS（多任务）

  ```
  并发和共享是多用户（多任务）OS 最基本的两个特征。
  ```

* [ ] 虚拟

  ```
  虚拟分为 "时分复用" 和 "空分复用"。
  时分复用：利用某设备为一用户服务的空闲时间，转去为其他用户服务，使设备得到充分利用。
  空分复用：将一个频率范围比较宽的信道划分为多个范围较窄的信道，
  ```

  

## 操作系统的主要功能

* [ ] 处理器管理功能

    ```
    
  1.进程控制：为了使作业并发执行，必须为每道作业创建一个或多个进程，并为其分配资源，当作业结束时，还得负责资源的回收。
  2.进程同步： a.进程互斥，指在访问临界资源时。 b.进程同步，对进程的执行顺序加以协调。
  3.进程通信.
  4.进程调度：
    a.作业调度，从后备队列中按照一定的算法选择除若干个作业，为他们分配所需要的资源。
    b.进程调度。从就绪队列中按照一定的算法选出一个进程，将处理机分配给他，为其设置运行现场，使其投入运行。         
  ```

* [ ] 存储器管理功能

  ```
  1.内存分配： 分配方式分为两种。
     a. 静态分配方式，每个作业的内存空间是在作业装入时确定的，在程序的运行期间不允许该作业申请新的空间，也不允许其移动。
     b. 动态分配方式，作业装入时也会申请空间，但在运行过程中也允许附加新的运行空间，也允许移动。
  2.内存保护：确保每道用户程序只在自己的内存空间内运行，彼此互不干扰，不允许用户程序访问操作系统的程序和数据，也不允许用户程序转移到非共享的其它用户程序内执行。
  3.地址映射：将逻辑地址转换为对用的物理地址。
  4.内存扩充：运用虚拟存储技术。
    a.请求调入功能：系统允许在仅装入部分用户程序和数据的情况下，便能启动程序，若发现需要的程序和数据未装入内存，便向OS发出请   求，由OS从磁盘中将所需部分调入内存。
    b.置换功能： 当发现没有足够的空间来装入需要调入的程序和数据时，便将一部分暂时不用的程序和数据调至磁盘上，腾出内存空间，
  ```

* [ ] 设备管理功能

  ```
  设备管理的主要任务：
   a. 完成用户进程提出的I/O请求，为用户进程分配所需的I/O设备，并完成指定的I/O操作.
   b. 提高CPU 和 I/O 设备的利用率，提高I/O 速度，方便用户使用 I/O 设备。
   
   1.缓冲管理
     为了有效的缓和CPU 和 I/O设备速度不匹配的矛盾，提高CPU的利用率。
   2.设备分配
   3.设备处理
  ```

* [ ] 操作系统结构

  ```
  1.无结构操作系统。 //功能的实现和高效率
  2.模块化操作系统。
  3.分层式结构。
    优点: 保证系统的正确性，自上而下的设计方式使所有设计中决定都是有序的。
          易扩充和易维护性，修改其中一个层次的结构，不会影响到其它层次。
    缺点: 缺点是效率较低，由于层次结构是分层单向依赖的，必须在每层中建立通信机制。 OS 每执行一个功能，都要穿越多个层次。
  ```

## 习题

1.设计现代OS的主要目标是什么？

```
有效性、方便性、可扩充性、开放性

方便性: 未配置OS的计算机系统是极其难用的，配置了OS可以极大的方便用户，使计算机变得易学易用。
有效性: 提供系统资源的利用率和系统的吞吐量。
可扩充性: 目前OS广泛采用微内核结构，方便添加新功能和模块。
开放性: 系统遵循世界标准规范。
```

2.OS的作用可表现在哪几个方面？

```
1.作为用户和计算机硬件系统之间的接口。
2.作为计算机资源的管理者。
3.实现了对计算机资源的抽象。
```

3.为什么说OS实现了计算机资源的抽象？

```
OS首先在裸机上覆盖一层I/O设备管理软件，实现了对计算机硬件操作的第一层次抽象；在第一层软件上在覆盖文件管理软件，实现了对硬件资源操作的第二次抽象。OS 通过在计算机硬件上安装多层系统软件，增强了系统功能，隐藏了对硬件操作的细节，由他们共同实现了对计算机资源的抽象。
```
